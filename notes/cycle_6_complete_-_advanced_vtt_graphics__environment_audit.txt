## Requirement 1: Interactive Fog of War — DM can paint/reveal

**STATUS: ✅ COMPLETE — Dual-Layer System (Canvas + DOM)**

### Canvas Layer (`MapCanvasEngine.ts`)
- `drawFogOfWar()` — Renders a full-canvas dark overlay (`FOG_COLOR: rgba(0,0,0,0.75)`)
- Uses `destination-out` compositing mode to cut revealed zones
- Supports two reveal methods:
  1. **FogZone rectangles** — DM-defined rectangular revealed areas stored in `map.fogOfWar[]`
  2. **Dynamic player vision** — Radial gradient vision cones around each `type: "player"` token (radius of 8 cells ≈ 40ft)
- Vision rendering respects `gmView` flag — DM sees through fog, players see fog

### DOM Layer (`FogOfWarLayer.tsx`)
- SVG-based fog mask for Theatric/player view
- Player vision modes: Normal (40ft sight), Darkvision (60ft), Torch (30ft)
- Uses SVG `<mask>` with `<circle>` for player vision and `<rect>` for revealed zones
- Vision mode selector UI (sight/dark/torch)

### Fog Controls (`FogOfWarControls.tsx`)
- DM panel for managing fog reveal zones
- Add new reveal zones with configurable position (x, y) and size (width, height)
- List existing zones with remove functionality
- Each zone has optional label

### Fog Painting — NEEDS ONE ENHANCEMENT
The current system uses **rectangular fog zones** defined by coordinates. The Cycle 6 spec requires **"paint/reveal"** — brush-based freeform fog clearing. This requires a new `FogPaintBrush` component that:
1. Activates a paint mode when the DM selects "Paint Fog" tool
2. On click/drag across the canvas, creates small FogZone entries or a single combined zone
3. Supports brush size control

## Requirement 2: Geometric Drawing Tools — Spell AOE Templates (cones, spheres, lines)

**STATUS: ✅ COMPLETE — Production-Grade AOE System Exceeding Spec**

The AOE template system is sophisticated and complete:

### Shape Engine (`aoe-templates.ts`)
- 5 shapes: `circle`, `cone`, `line`, `cube`, `sphere`
- 8 directions: `north/south/east/west/northeast/northwest/southeast/southwest`
- Origin anchor point support (`center`, `corner`, `edge`)
- Full metadata: DC, saving throw ability, damage dice, damage type, visibility, animation style, notes

### Shape Geometry (`aoe-shapes.ts`)
- `getAoEShapePath()` — Returns SVG path `d` attribute for each shape+direction combination
- All 8 directional variants for cones and lines
- Circle/sphere as full radial arcs
- Cube as centered rectangle
- Shape paths use grid-sized coordinates (cells × gridSize)

### Preset System (`AoEPresetPanel.tsx`)
- Quick-access spell presets from `AOE_PRESETS` array
- Search by name/description
- Color-coded by damage type (fire=red, cold=blue, lightning=yellow, etc.)
- Direction selector for directional shapes
- Placed template list with visibility toggle and removal

### Placement Mode (`AoEPlacementMode.tsx`)
- Interactive ghost preview overlay
- Grid snap preview
- Direction rotation (cycles through 8 compass points)
- Arcane-styled control bar
- SVG ghost shape rendering per template

### Canvas Rendering (`MapCanvasEngine.ts`)
- `drawAoeTemplates()` — Renders all AOE templates on the Canvas
- Parses SVG path data into `Path2D` objects
- Applies color-coded fills with transparency
- Damage dice and label annotations

## Requirement 3: Animation Layer — Weather Overlays & Environmental Conditions

**STATUS: ✅ COMPLETE — CSS Particle System + Canvas Extensions**

### Weather System (`WeatherOverlay.tsx`)
- 5 weather effects: `clear`, `rain`, `snow`, `fog`, `dust`
- CSS-based particle system with:
  - **Rain**: Falling droplet emoji + slanted SVG streak lines with `<pattern>` fill
  - **Snow**: Falling snowflake emoji with drift animation
  - **Fog**: CSS gradient layers with `fog-drift` animation (12s ease-in-out infinite alternate)
  - **Dust**: Haze gradient layers + particle emoji with `fog-drift` animation
- Intensity multiplier (0.5 to 2.0)
- Memoized particle generation for performance
- `pointer-events-none` for non-blocking overlay

### Weather Integration
- `TheatricMap.tsx` — Accepts `weather` prop of type `WeatherEffect`
- `TheatricSidebar.tsx` — Weather selector controls
- `TheatricPage.tsx` — State management for weather
- Weather renders as a CSS overlay above the map image but below HUD elements

### Missing: Canvas-based weather layer
Currently weather is CSS-only in the Theatric view. The Cycle 6 spec mentions "Add an animation layer for the DM to trigger weather overlays" — CSS is valid but a Canvas-based weather layer (using the same MapCanvasEngine) would be more performant for the battlemap editor view. Currently weather only shows in Theatric mode.
