### Requirement 1: Canvas/WebGL approach, completely separated from React HUD

**STATUS: ✅ COMPLETE — Premium HTML5 Canvas Architecture**

The codebase has a **pure Canvas engine** (`MapCanvasEngine.ts`) that is completely separated from the React DOM:

| File | Purpose |
|------|---------|
| `lib/canvas/MapCanvasEngine.ts` | **Core Canvas renderer** — Renders map background (image or procedural grid), grid lines (major/minor), fog of war, token circles/images, HP bars, status markers, movement range circles, AOE templates, freehand drawings, selection pulse animations. ### Requirement 3: Draggable tokens for PCs and enemies that broadcast X/Y coordinates to Firebase

**STATUS: ✅ COMPLETE — Token System with Drag-and-Drop + Firebase Sync**

**Token System:**
- `AddTokenForm.tsx` — Full token creation form (label, type, position, color, size, speed, initiative, HP, image)
- `MapToken` type — Supports player/enemy/npc/custom types, HP tracking, status markers, initiative, imageUrl
- `CanvasMapView.tsx` — `handleMouseDown/Move/Up` implements full drag-and-drop:
  - Click on token → selects it (`onTokenClick`)
  - Drag token → moves with mouse
  - Release → fires `onTokenDragEnd(tokenId, gridX, gridY)` with final coordinates
- `MapCanvasEngine.drawTokens()` — Renders tokens as colored circles with image support, HP bars, status dots, labels, selection pulse
- `MapCanvasEngine.getTokenAt()` — Hit-testing to find token at mouse position (reversed iteration for z-order)

**Firebase Sync:**
- `normalized-firebase-service.ts` — Already has `normalizedTokens` using `createTokenStub` pattern
- `normalizedSync.listenTokens()` — Subscribes to real-time token updates for a specific map
- `normalizedSync.unlistenTokens()` — Cleanup
- Path: `campaigns/{cid}/maps/{mid}/tokens/{tid}`
- `BattleMaps.tsx` — Tokens are pushed/updated via `updateBattleMap()` which syncs the entire map doc to Firebase

### Requirement 4: Broadcast token coordinates via Firebase

**STATUS: ⚠️ NEEDS WIRING — The token drag callback exists, but needs a dedicated push**

The `onTokenDragEnd` callback from `CanvasMapView` provides `(tokenId, gridX, gridY)`. The `MapEditor.tsx` receives this but needs a hook that:
1. Captures the drag-end event
2. Calls `normalizedTokens.push(campaignId, mapId, updatedToken)` to broadcast the new position
3. All connected clients receive the update via `normalizedTokens.listenAll()`

This is a **small integration task** — the infrastructure is fully built, just needs the final wire-up.

## What Cycle 5 Actually Requires (Surgical Integration)

Only one integration is missing — wire the token drag-end event to Firebase. Add to `MapEditor.tsx`:

```typescript
const handleTokenDragEnd = useCallback((tokenId: string, gridX: number, gridY: number) => {
  const updatedTokens = map.tokens.map((t) =>
    t.id === tokenId ? { ...t, x: gridX, y: gridY } : t
  );
  onUpdate({ tokens: updatedTokens });
  // Firebase broadcast
  if (campaignId && firebaseReady) {
    normalizedTokens.push(campaignId, map.id, {
      id: tokenId,
      data: updatedTokens.find(t => t.id === tokenId)!,
      updatedAt: Date.now(),
      updatedBy: 'dm'
    });
  }
}, [map, onUpdate, campaignId, firebaseReady]);
```

Everything else for Cycle 5 is **already production-grade**.
